# Routing and Invocation Examples

These examples illustrate semantic intent. Do not treat their wording as a keyword list.

## Use: Seed

The user has an early concept for an AI service for independent restaurants and wants help turning it into a viable project. The agent should identify the Seed mode, clarify whose operational problem matters, separate evidence from enthusiasm, and stop after Stage 1 for confirmation.

## Use: Stress-test

The user plans to replace a manual approval process with an automated workflow and wants an assessment before implementation. The agent should identify unverified adoption, exception-handling, and governance assumptions without pretending the proposal has already failed.

## Use: Diagnose

The user launched an internal knowledge assistant, but usage fell after the first week. The agent should restate the evidence, distinguish adoption, answer quality, discoverability, incentives, and measurement explanations, then link each later recommendation to a specific finding.

## Do not use: Direct execution

The user supplies an approved specification and asks the agent to update one configuration file. Unless the request also asks whether the direction is sound, execute the scoped change using the relevant implementation workflow.

## Do not use: Isolated bug

The user provides a reproducible stack trace and asks why one function throws. Use the agent's debugging workflow. Escalate to Foolish Think only if evidence suggests the bug exposes a broader project or architecture assumption.

## Ambiguous

The user says only, “Help with my project.” Ask one intent question. Do not launch all three stages merely because a project was mentioned.
